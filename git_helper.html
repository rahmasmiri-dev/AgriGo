📘 Guide Complet : Maîtriser Git et le Terminal Windows

Ce guide récapitule les commandes essentielles pour naviguer dans votre système et collaborer efficacement sur un projet Git à plusieurs (5 collaborateurs).

💻 1. Commandes Système (Windows)

Pour naviguer dans vos dossiers et visualiser vos fichiers. cd D:\2BI\s2\agl\git

cd [chemin] : Change le répertoire courant.

D:

D: : Change de lecteur pour passer sur la partition D.

dir

dir : Liste les fichiers et dossiers présents dans le répertoire actuel.

tree /f

tree /f : Affiche l'arborescence complète avec tous les fichiers.

⚙️ 2. Configuration et Vérification Git

À faire lors de la première installation. git --version

Vérifie si Git est installé et affiche sa version.

git config --global user.name "votre_nom" git config --global user.email "votre_email@gmail.com"

Définit votre identité pour tous vos dépôts.

git config --list

Affiche tous les paramètres de configuration actuels.

🌿 3. Gestion du Dépôt et des Branches

Pour organiser le travail d'équipe en parallèle. git clone https://github.com/user/NomProjet.git

Télécharge une copie distante du projet sur votre ordinateur.

git branch feature-melak

Crée une nouvelle branche pour vos développements.

git branch -a

Liste toutes les branches (locales en blanc/noir et distantes en rouge).

git switch feature-melak

Bascule sur la branche spécifiée pour commencer à travailler.

git merge feature-melak

Fusionne les modifications de la branche "feature-melak" dans la branche actuelle (souvent main).

🔄 4. Cycle de Modification (Flux de travail)

Les commandes répétées quotidiennement. git status

Affiche l'état des fichiers (modifiés, suivis ou non).

git add .

Ajoute toutes les modifications à la zone de préparation (staging).

git commit -m "Description claire du changement"

Enregistre vos modifications dans l'historique local.

☁️ 5. Synchronisation avec GitHub

Envoyer et recevoir du code du serveur distant. git fetch --all

Récupère les mises à jour de toutes les branches du serveur sans modifier vos fichiers.

git pull origin main

Télécharge et fusionne les nouveautés du serveur dans votre branche actuelle.

git push origin feature-melak

Envoie votre branche locale vers GitHub.

📜 6. Consultation de l'Historique

Pour voir qui a fait quoi. git log --oneline --graph --all

Affiche une version visuelle et simplifiée de l'historique de toutes les branches.

git log --author="Rahma"

Filtre les commits pour voir uniquement ceux de Rahma.

git shortlog -s -n

Résumé du nombre de commits par auteur (classement des contributeurs).

❌ Erreurs courantes à éviter

  • git--version : Attention à l'espace ! La commande est git --version.
  • fetch ou get status : N'oubliez jamais le mot git devant (ex: git fetch).
  • Ne travaillez jamais directement sur la branche main.
  • Faites un git pull avant de commencer votre journée pour éviter les conflits.

🏁 Conclusion

En suivant ce flux, vous garantissez un projet organisé :

  • ✔️ Un historique propre grâce aux messages de commit clairs.
  • ✔️ Aucune perte de code grâce aux branches individuelles.
  • ✔️ Une synchronisation parfaite avec GitHub.
